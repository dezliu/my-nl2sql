# 管理后台扩展 + SSE 实时流式输出

> 文档位置：`doc/plan/管理后台数据管理扩展.md`

## 概述

本迭代合并两项能力：

1. **管理后台扩展**：表元数据录入、业务数据/FAQ、SQL 模板与自动推荐审核、向量入库开关
2. **SSE 实时流式**：用户端改用 `POST /api/ask/stream`，实现 LLM token 级实时输出

---

## Part A：SSE 实时流式

### 后端

| 文件 | 改动 |
|------|------|
| `backend/api/sse.py` | `POST /api/ask/stream` → `text/event-stream` |
| `backend/api/session_manager.py` | `push_event`、`make_queue_emitter`、`sse_event_generator` |
| `backend/llm/service.py` | `astream()` + 缓存命中逐字推送 |
| `backend/graph/workflow.py` | `event_emitter` + `LLM_TOKEN` 推送 |
| `backend/api/main.py` | 挂载 SSE 路由 |

### SSE 事件类型

| event | 说明 |
|-------|------|
| `INTENT` | 意图识别完成 |
| `RAG_CHUNK` | 检索结果 |
| `LLM_TOKEN` | LLM 增量 token `{ role, delta }` |
| `SQL` | SQL 定稿 |
| `RESULT` | 查询结果 |
| `SUMMARY` | 总结定稿 |
| `ERROR` | 错误 |
| `DONE` | 结束 |

### 前端

| 文件 | 改动 |
|------|------|
| `apps/web/lib/sse.ts` | `fetch` + `ReadableStream` 解析 SSE |
| `apps/web/app/page.tsx` | 移除 GraphQL Subscription，实时累加 token |

环境变量：`NEXT_PUBLIC_API_URL=http://localhost:8000`

GraphQL `askQuestion` / `askStream` 保留兼容，用户端默认走 SSE。

## Part B：管理后台扩展

### 数据库迁移 Alembic `002`

- `table_metadata.is_indexed`
- `sql_templates.is_indexed`
- `business_glossary.is_indexed`
- 新表 `knowledge_entries`（FAQ/知识）
- 新表 `template_recommendations`（status: pending/approved/rejected）

```bash
alembic upgrade head
```

### 索引管道

`backend/rag/indexer.py`：

- `index_table` / `index_template` / `index_glossary_term` / `index_fk` / `index_knowledge`
- `unindex_by_source`
- `backend/rag/index_ops.py`：`index_item` / `unindex_item` GraphQL 入口

### 模板自动推荐

`backend/services/template_recommender.py`：

- 条件：`message_sql.was_executed=true` + `rag_quality_scores.score >= 0.8`
- Celery：`scan_template_candidates`

### GraphQL CRUD

见 `backend/api/admin_graphql.py`，包含：

- 元数据：表/列/FK CRUD
- 业务：术语 + 知识库 CRUD
- 模板 CRUD + 推荐审核
- `indexItem` / `unindexItem` / `testRagSearch`

### 管理页

| 路由 | 功能 |
|------|------|
| `/metadata` | 表/列/FK CRUD + 向量开关 |
| `/business` | 术语 + FAQ/知识 |
| `/templates` | 模板 CRUD + 推荐审核 |
| `/rag-search` | 向量检索测试 |

## Part D：风险

- SSE 长连接需处理客户端断开（cancel workflow task）
- token 流与缓存命中策略需一致
- Qdrant 单条 index 与全量 rebuild 需 upsert 去重（已实现 unindex 后再 index）

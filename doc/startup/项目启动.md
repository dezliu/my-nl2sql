# 项目启动

> 文档位置：`doc/startup/项目启动.md`

## 前置要求

- Docker & Docker Compose
- Python 3.11+
- Node.js 18+
- OpenAI API Key（或兼容接口）

## 一、启动基础设施

```bash
docker compose up -d
```

启动以下服务：

| 服务 | 地址 | 说明 |
|------|------|------|
| MySQL | `localhost:3306` | 用户 `nl2sql` / 密码 `nl2sql` / 库 `nl2sql` |
| Qdrant | `http://localhost:6333` | 向量数据库 |
| Redis | `localhost:6379` | Celery 消息队列 |

## 二、配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`，至少配置：

```env
OPENAI_API_KEY=sk-your-key-here
OPENAI_MODEL=gpt-4o-mini
DATABASE_URL=mysql+aiomysql://nl2sql:nl2sql@localhost:3306/nl2sql
QDRANT_URL=http://localhost:6333
REDIS_URL=redis://localhost:6379/0
CELERY_BROKER_URL=redis://localhost:6379/1
CELERY_RESULT_BACKEND=redis://localhost:6379/2
```

## 三、安装后端依赖

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
```

或使用 Makefile：

```bash
make install
```

## 四、数据库迁移与种子数据

```bash
alembic upgrade head
python -m backend.scripts.seed
```

或使用 Makefile：

```bash
make migrate
make seed
```

种子数据会创建：Demo 数据源、users/orders 表元数据、FK 关系、SQL 模板、业务术语及 9 角色默认 Prompt。

## 五、启动后端 API

```bash
uvicorn backend.api.main:app --reload --port 8000
```

或使用 Makefile：

```bash
make api
```

验证：

- 健康检查：http://localhost:8000/health
- GraphQL Playground：http://localhost:8000/graphql

## 六、启动 Celery Worker

新开终端：

```bash
source .venv/bin/activate
celery -A backend.workers.tasks worker --loglevel=info
```

或使用 Makefile：

```bash
make worker
```

Worker 负责 RAG chunk 异步评分；评分低于 0.6 时写入告警表。

## 七、启动前端

### 安装依赖（首次）

```bash
npm install
```

### 用户端（:3000）

```bash
npm run dev:web
# 或
make web
```

访问：http://localhost:3000

### 管理后台（:3001）

```bash
npm run dev:admin
# 或
make admin
```

访问：http://localhost:3001

前端默认 GraphQL 地址：

- HTTP：`http://localhost:8000/graphql`
- WebSocket：`ws://localhost:8000/graphql`

可在各 app 下通过环境变量覆盖：

```env
NEXT_PUBLIC_GRAPHQL_URL=http://localhost:8000/graphql
NEXT_PUBLIC_GRAPHQL_WS_URL=ws://localhost:8000/graphql
```

## 八、完整启动顺序（速查）

```bash
# 终端 1：基础设施
docker compose up -d

# 终端 2：后端（首次需 migrate + seed）
source .venv/bin/activate
alembic upgrade head
python -m backend.scripts.seed
uvicorn backend.api.main:app --reload --port 8000

# 终端 3：Celery
source .venv/bin/activate
celery -A backend.workers.tasks worker --loglevel=info

# 终端 4：前端
npm install
npm run dev:web      # 用户端
npm run dev:admin    # 管理后台（另开终端）
```

## 九、常用 Makefile 命令

| 命令 | 说明 |
|------|------|
| `make up` | 启动 Docker 基础设施 |
| `make down` | 停止 Docker 基础设施 |
| `make migrate` | 执行 Alembic 迁移 |
| `make seed` | 写入种子数据 |
| `make api` | 启动 FastAPI |
| `make worker` | 启动 Celery Worker |
| `make web` | 启动用户端前端 |
| `make admin` | 启动管理后台 |
| `make test` | 运行后端单元测试 |

## 十、首次使用建议流程

1. 启动全部服务（见上文）
2. 打开管理后台 http://localhost:3001
3. 进入 **元数据** 页，选择 Demo 数据源，点击 **重建 RAG 索引**
4. 打开用户端 http://localhost:3000，选择 Demo 数据源
5. 输入问题，例如：`查询每个用户的订单总数`
6. 可选开启 **深度思考 (ReAct)** 或切换执行模式

## 十一、运行测试

```bash
source .venv/bin/activate
pytest backend/tests -q
# 或
make test
```

## 十二、常见问题

**MySQL 连接失败**  
确认 `docker compose ps` 中 mysql 状态为 healthy，等待约 10–30 秒后重试迁移。

**Qdrant 连接失败**  
确认 6333 端口可访问；首次启动会自动创建 collection。

**GraphQL Subscription 无流式输出**  
确认前端 WebSocket 地址正确，且后端 uvicorn 支持 WebSocket（默认支持）。

**OpenAI 调用失败**  
检查 `.env` 中 `OPENAI_API_KEY` 是否有效。

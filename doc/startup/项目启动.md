# 项目启动

> 文档位置：`doc/startup/项目启动.md`

## 前置要求

- Docker & Docker Compose
- Python 3.11+
- Node.js 18+
- OpenAI API Key（或兼容接口）

## 一、启动基础设施

```bash
docker compose up -d
```

> **网络说明**：`docker-compose.yml` 已配置国内镜像源 `docker.1ms.run`，避免 Docker Hub (`registry-1.docker.io`) 连接超时。若你使用其他镜像加速器，可自行替换 compose 中的镜像前缀。

启动以下服务：

| 服务 | 地址 | 说明 |
|------|------|------|
| MySQL | `localhost:3307` | 用户 `nl2sql` / 密码 `nl2sql` / 库 `nl2sql`（3306 被占用时使用 3307） |
| Qdrant | `http://localhost:6333` | 向量数据库 |
| Redis | `localhost:6379` | Celery 消息队列 |

## 二、配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`，至少配置：

```env
# 官方 OpenAI（Key 以 sk- 开头）：
# OPENAI_API_KEY=sk-your-key-here
# OPENAI_MODEL=gpt-4o-mini

# 智谱 AI 等 OpenAI 兼容接口（推荐国内）：
OPENAI_API_KEY=your-zhipu-api-key
OPENAI_BASE_URL=https://open.bigmodel.cn/api/paas/v4
OPENAI_MODEL=glm-4-flash
DATABASE_URL=mysql+aiomysql://nl2sql:nl2sql@localhost:3307/nl2sql
QDRANT_URL=http://localhost:6333
REDIS_URL=redis://localhost:6379/0
CELERY_BROKER_URL=redis://localhost:6379/1
CELERY_RESULT_BACKEND=redis://localhost:6379/2
```

## 三、安装后端依赖

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
```

或使用 Makefile：

```bash
make install
```

## 四、数据库迁移与种子数据

```bash
alembic upgrade head
python -m backend.scripts.seed
```

> **说明**：后端 API 启动时会自动执行 `alembic upgrade head`，确保 schema 与代码一致。首次部署或升级后仍建议手动执行一次上述命令。

或使用 Makefile：

```bash
make migrate
make seed
```

种子数据会创建：Demo 数据源、users/orders 表元数据、FK 关系、SQL 模板、业务术语及 9 角色默认 Prompt。

**升级 Prompt（防幻觉版本）**：若代码更新了 `DEFAULT_PROMPTS` 且数据库中已有旧版 Prompt，需执行：

```bash
make upgrade-prompts
# 或
python -m backend.scripts.upgrade_prompts
```

会为 `sql_generator`、`react_reasoner`、`query_expander`、`sql_safety` 等角色创建新版本并自动激活（含中文回答默认与角色锁定策略）。

> **迁移 002**：执行 `alembic upgrade head` 后会新增 `knowledge_entries`、`template_recommendations` 表及 `is_indexed` 字段。

## 五、启动后端 API

```bash
uvicorn backend.api.main:app --reload --port 8000
```

或使用 Makefile：

```bash
make api
```

验证：

- 健康检查：http://localhost:8000/health
- GraphQL Playground：http://localhost:8000/graphql
- SSE 流式问答：`POST http://localhost:8000/api/ask/stream`

## 六、启动 Celery Worker

新开终端：

```bash
source .venv/bin/activate
celery -A backend.workers.tasks worker --loglevel=info
```

或使用 Makefile：

```bash
make worker
```

Worker 负责 RAG chunk 异步评分；评分低于 0.6 时写入告警表。另可手动触发模板推荐扫描：

```bash
celery -A backend.workers.tasks call scan_template_candidates
```

## 七、启动前端

### 安装依赖（首次）

```bash
npm install
```

### 用户端（:4000）

```bash
npm run dev:web
# 或
make web
```

访问：http://localhost:4000

**停止与继续生成**（用户端聊天）：

- 生成过程中点击「停止」：中止当前流式输出，**原问题自动填回输入框**
- 未修改问题再次点击「继续」（或发送）：从上次完成的**工作流节点**接着执行，不会从意图识别重头开始
- 若修改了输入内容再发送，则视为新问题，放弃可续跑会话
- 限制：续跑粒度为 LangGraph 节点（停在 SQL 生成中途时，恢复会重跑该节点）；checkpoint 在进程内存中，**重启 API 后无法续跑**

### 管理后台（:4001）

```bash
npm run dev:admin
# 或
make admin
```

访问：http://localhost:4001

管理后台新增页面：**元数据**（CRUD + 向量开关）、**业务数据**（术语/FAQ）、**SQL 模板**（推荐审核）、**向量检索**（RAG 测试）。

前端默认 GraphQL 地址：

- HTTP：`http://localhost:8000/graphql`
- WebSocket：`ws://localhost:8000/graphql`

可在各 app 下通过环境变量覆盖：

```env
NEXT_PUBLIC_GRAPHQL_URL=http://localhost:8000/graphql
NEXT_PUBLIC_GRAPHQL_WS_URL=ws://localhost:8000/graphql
NEXT_PUBLIC_API_URL=http://localhost:8000
```

用户端问答默认使用 SSE（`NEXT_PUBLIC_API_URL`），GraphQL 仍用于数据源列表等 Query。

## 八、完整启动顺序（速查）

```bash
# 终端 1：基础设施
docker compose up -d

# 终端 2：后端（首次需 migrate + seed）
source .venv/bin/activate
alembic upgrade head
python -m backend.scripts.seed
uvicorn backend.api.main:app --reload --port 8000

# 终端 3：Celery
source .venv/bin/activate
celery -A backend.workers.tasks worker --loglevel=info

# 终端 4：前端
npm install
npm run dev:web      # 用户端
npm run dev:admin    # 管理后台（另开终端）
```

## 九、常用 Makefile 命令

| 命令 | 说明 |
|------|------|
| `make up` | 启动 Docker 基础设施 |

| `make migrate` | 执行 Alembic 迁移 |
| `make seed` | 写入种子数据 |
| `make upgrade-prompts` | 发布并激活防幻觉 Prompt 新版本 |
| `make api` | 启动 FastAPI |
| `make worker` | 启动 Celery Worker |
| `make web` | 启动用户端前端 |
| `make admin` | 启动管理后台 |
| `make test` | 运行后端单元测试 |

## 十、首次使用建议流程

1. 启动全部服务（见上文）
2. 打开管理后台 http://localhost:4001
3. 进入 **元数据** 页，录入表/列/FK，或使用 **重建 RAG 索引** 全量入库
4. 可选：在 **业务数据** / **SQL 模板** 页维护术语、FAQ、模板
5. 打开用户端 http://localhost:4000，选择 Demo 数据源
6. 输入问题，例如：`查询每个用户的订单总数`（回答将以 SSE token 流实时显示）
7. 可选开启 **深度思考 (ReAct)** 或切换执行模式

## 十二、常见问题

**MySQL 连接失败**  
确认 `docker compose ps` 中 mysql 状态为 healthy，等待约 10–30 秒后重试迁移。

**端口 3306 已被占用**  
本机已有 MySQL 占用 3306 时，可停止冲突服务，或在 `docker-compose.yml` 中将 `3306:3306` 改为 `3307:3306`，并同步修改 `.env` 中的 `DATABASE_URL` 端口。

**Docker Hub 拉取超时（context deadline exceeded）**  
说明无法访问 `registry-1.docker.io`。项目已默认使用 `docker.1ms.run` 镜像源；若仍失败，请检查网络或配置 Docker Desktop 镜像加速器。

**Qdrant 连接失败**  
确认 6333 端口可访问；首次启动会自动创建 collection。

**用户端无流式输出**  
确认 `NEXT_PUBLIC_API_URL=http://localhost:8000` 且后端 SSE 路由可访问。用户端已改用 `POST /api/ask/stream`，不再依赖 GraphQL Subscription。

**GraphQL Subscription 无流式输出（管理端/兼容路径）**  
确认前端 WebSocket 地址正确，且后端 uvicorn 支持 WebSocket（默认支持）。

**OpenAI 调用失败（401 invalid_api_key）**  
- 官方 OpenAI Key 应以 `sk-` 开头；智谱 Key 格式为 `32位hex.16位字符`，需配 `OPENAI_BASE_URL=https://open.bigmodel.cn/api/paas/v4` 且模型改为 `glm-4-flash` 等智谱模型名  
- 其他第三方兼容接口同理：同时配置 `OPENAI_API_KEY` + `OPENAI_BASE_URL` + 该平台支持的 `OPENAI_MODEL`  
- 修改 `.env` 后需重启后端 `make api`
